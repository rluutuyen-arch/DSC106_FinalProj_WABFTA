Design & Process

Progress

So far, our team has built the initial structure of our explorable explanation webpage, which focuses on how sea surface temperature shifts across the Western Hemisphere over a 72-hour period. We started by working with GOES-16 NOAA satellite sea surface temperature data and preprocessing it into a format that could be displayed interactively in D3.js. Since the original data is spatial, we transformed the coordinate values into x/y screen positions so they could be accurately rendered on the webpage, while also experimenting with different scaling methods to make the map clearer and easier to interpret. We also added a continent outline to give viewers more geographic context, so the temperature patterns are not shown as abstract pixels floating on a blank canvas.

In the current prototype, we have at least one working visualization with interaction, including a time-based animation/slider that lets users move through the 72 hourly frames. We also added supporting charts, including a mean sea surface temperature line chart and a temperature distribution histogram, so users can connect the spatial patterns on the map with broader statistical trends. Another major update is that we added zoom and pan functionality, which allows viewers to explore specific regions of the Western Hemisphere in more detail instead of only viewing the full map at once. Overall, our prototype now has the basic webpage layout, main visualization, interaction structure, and supporting explanatory elements needed for the final explorable explanation.

Challenges

The most challenging part of the project to design has been converting the satellite data into a clear and usable interactive visualization. Because the raw data is based on geographic/satellite coordinates, we had to figure out how to transform those values into x/y positions that would display correctly in the browser. This was especially challenging because small changes in scaling could make the visualization look distorted, too zoomed in, or too difficult to interpret. Adding the continent outline also required us to think carefully about alignment, since the outline needed to match the temperature pixels closely enough to provide context without distracting from the actual data.

Another challenge has been balancing visual complexity with performance and readability. The dataset contains many temperature points across many time frames, so we had to design the page in a way that allows smooth interaction without overwhelming the viewer. We also had to decide which interactions actually help explain the data, rather than adding features just for the sake of interactivity. Moving forward, we plan to refine the annotations, explanatory text, and visualization so the webpage feels more like an explorable explanation instead of just a dashboard, which matches the final project goal of using interactive visualizations to explain a complex subject.
